 Python - Exception Handling


Exception handling is very important in programming. It allows an executing program to not terminate immedialtely after an exception occurs at any point.Let us see how to handle exception in Python. As I have not decribed the whole concept here that is why I have commented the lines. You can also visit Errors and Execptions and read it more clearly.



  """
      created_at       : 09 Feb 2017.
      updated_at       : 11 Feb 2017.
      aim of script    : To understand the concept of exception handling in Python.
      coded by         : Rishikesh Agrawani.
      Python version   : 2.7.12
  """
  a = 3
  b = 4
  name = 'Randy'

  try:
      print a+b           #It will get executed
  except:
      print "Error while adding 2 numbers"

  try:
      print a+name        #It raises TypeError

  try:

Till now, we only focused on the try-except blocks(clauses). Apart from this, try-except statement has 2 other optional clauses. These are else clause & finallly clause.Let us see these clauses in the following examples

The main usage of else block is to put the error free code(that follows the code inside try clause) inside it. In this way we can reduce the number of lines present inside the try clause.

The finally block is used to define the clean up codes. For example, closing the opened files, releasing the netwok connections etc.

        
 #Example 7 - UnboundLocalError (try-except-else-finally clauses)

  try:
      n1 = 10                              #Assigning a value to any variable a way of defining/redefining the variable in that scope
      n2 = 5
      print n1, n2                         #It will be printed
  except:
      print "An error occured"
  else:
      print "No error\n"                   #It will get executed

  try:
      def print_num():
          n1+=1
          print n1
      print_num()                          #It will raise an exception
  except UnboundLocalError as unbound_err:
      print "Either you are trying to access any local variable defined in other suite or an undefined variable."
      print unbound_err                    #local variable 'n1' referenced before assignment
  else:
      print "Just silent"                  #It won't get executed

In previous examples, we saw that we are handling only single exception in each example. So what about there's a possibility of multiple exceptions. Python also has the great technique for this. Let us see how to specify multiple exceptions in except clause. 

        
 #Example 8 - KeyboardInterrupt, ValueError, ZeroDivisionError

  def test_func(b,s):
      try:
          d = raw_input('\n\nINPUT : Enter any info Or press Ctrl+c or Command+c to raise KeyboardInterrupt Exception : ')
          a = 10
          print a/b                             #If b equals 0

          c = int(s)                            #It will raise an exception, if scontains non-digits
      except (ValueError,KeyboardInterrupt,ZeroDivisionError):
          print "\nMSG : Either ValueError, KeyboardInterrupt, ZeroDivisionError Exception occured... "
      else:
          print "No error occured"
      finally:
          print "This is the last statement(residing inside finally clause) to be executed from try-except-else-finally clauses"

  test_func(5,"1235")                    #It will not raise any exception, so else clause will execute if you don't press Ctrl+c/Command+c
  test_func(0,"1235")                    #ZeroDivisionError will be raised by this statement if you don't press Ctrl+c/Command+c
  test_func(5,"1235abc")                 #ValueError will be raised by this statement if you don't press Ctrl+c/Command+c

In the previous example, it is not clear that which function call is raising which exception because we have defined the Exceptions collectively inside a tuple of except statement. Let us find the type of exceptions separately and generate the meaningful messages to the user. For the sake of simplicity, I am using the previous example here.

        
 #Example 8 - KeyboardInterrupt, ValueError, ZeroDivisionError (modified)

  def test_func(b,s):
      try:
          d = raw_input('\n\n>>> Enter any info Or press Ctrl+c or Command+c to raise KeyboardInterrupt Exception : ')
          a = 10
          print a/b                       #If b equals 0

          c = int(s)                      #It will raise an exception, if s contains non-digits
      except ValueError:
          print "\nMSG : ValueError Exception occured... "
      except KeyboardInterrupt:
          print "\nMSG : KeyboardInterrupt Exception occured... "
      except ZeroDivisionError:
          print "\nMSG : ZeroDivisionError Exception occured... "
      else:
          print "No error occured"
      finally:
          print "This is the last statement(residing inside finally clause) to be executed from try-except-else-finally clauses"

  test_func(5,"1235")                      #It will not raise any exception, so else clause will execute if you don't press Ctrl+c/Command+c
  test_func(0,"1235")                      #ZeroDivisionError will be raised by this statement
  test_func(5,"1235abc")                   #ValueError will be raised by this statement
